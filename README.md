# «Построение графиков и оптимизация вывода (Алгоритм Дугласа-Пекера)»

Цель работы: Реализация системы отображения графиков функций на языке C++ с использованием фреймворка Qt, 
поддержка масштабирования, перемещения осей и оптимизация количества точек при отрисовке.
### 2. Описание реализации

#### 2.1. Система координат (Класс `Axis`)
Класс отвечает за логику отображения математических значений на экранные пиксели.
*   **Методы преобразования**: `realCoordToPixel` и `pixelCoordToReal` позволяют переходить от координат функции к координатам виджета.
*   **Управление**: Реализованы методы `scroll` (сдвиг границ `mLower`/`mUpper`) и `scale` (умножение диапазона на коэффициент относительно точки курсора).

#### 2.2. График (Класс `PolylineGraph`)
Класс управляет данными кривой.
*   **Генерация**: В работе реализовано построение трохоиды/циклоиды по параметрическим уравнениям:
    *   $x = r \cdot t - h \cdot \sin(t)$
    *   $y = r - h \cdot \cos(t)$
*   **Отрисовка**: Использует `QPainter::drawPolyline` для эффективного вывода массива точек.

### 3. Описание алгоритма оптимизации (Дугласа-Пекера)

Для минимизации нагрузки на графическую подсистему при работе с большим количеством точек применен рекурсивный алгоритм упрощения линий.

**Принцип работы:**
1.  Алгоритм рассматривает отрезок между начальной и конечной точками кривой.
2.  Среди всех промежуточных точек ищется та, которая максимально удалена от этого отрезка (перпендикулярное расстояние).
3.  Если найденное расстояние превышает установленный порог точности, кривая разбивается на две части в этой точке, и процедура повторяется рекурсивно.
4.  Если же максимальное отклонение меньше порога, все промежуточные точки на данном участке отбрасываются как незначимые для визуального восприятия.

Это позволяет сохранять характерные изгибы функции, значительно уменьшая количество сегментов, которые необходимо обрабатывать графическому процессору.

### 4. Реализованный функционал
*   Генерация массивов данных на основе математических формул.
*   Автоматическое масштабирование и позиционирование осей.
*   Интерактивное взаимодействие с графиком (зум колесиком мыши, перемещение захватом).
*   Переключение режима оптимизации для сравнения визуального качества и количества используемых точек.

Использование алгоритма Дугласа-Пекера показало свою эффективность в задачах визуализации, позволяя сохранять высокую детализацию графиков при значительном снижении затрат вычислительных ресурсов.
